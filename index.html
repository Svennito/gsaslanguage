gsaslanguage

This script language allows to automate the analysis of diffraction data using the GSAS Rietveld package. Githib is the new home of gsaslanguage!

View on GitHub Download .zip Download .tar.gz

gsaslanguage was developed out of the necessity to both automate and document Rietveld analysis of diffraction data. A script language accomplishes both:

  • It allows to analyze many datasets with a pre-defined and reproducible Rietveld strategy
  • The source code of the script and the generated PDF files document what the refinement strategy, i.e. the parameter turn on sequence, was.

The HIPPO neutron time-of-flight diffractometer at LANSCE collects data in either five or 45 histograms (for texture analysis), so one basically always has to deal with a multi-pattern data analysis. With gsaslanguage that is done in a few minutes. Because HIPPO is also fairly close to the intense LANSCE neutron source (~ 8.9m), we can collect meaningful data in around a minute in many cases, which allows us to study e.g. phase transformation kinetics. In cases like this, we collect a few hundred runs for a given sample, see for instance our paper on UC2. gsaslanguage is also useful for parametric studies, e.g. determination of the crystal structure as a function of temperature like various MAX phases with Michel Barsoum and his group (see e.g. this paper).

While I use Luca Lutterotti's MAUD for most texture analysis because it implements the more accurate WIMV method, in some cases I still use GSAS and the spherical harmonics ODF model. In cases like that, gsaslanguage comes in handy to control the analysis of close to 100 histograms with over 1,000 parameters.

Please reference my paper when you use gsaslanguage for your project. Please let me know if there are problems - I am happy to help.
